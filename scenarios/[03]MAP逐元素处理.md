# 03 — MAP 逐元素处理：对数组中的每个元素施加变换

> **适用版本**：Microsoft 365 / Excel 2021+

## 简介

`MAP` 接受一个或多个**等尺寸数组**以及一个 `LAMBDA`，对每个位置的元素执行变换，返回同尺寸的结果数组。它类似编程语言中的 `map` / `Select`。

语法：`=MAP(array1, [array2, ...], LAMBDA(elem1, [elem2, ...], 表达式))`

---

## 示例 1 ⭐ — 批量去空格并转大写

```excel
=MAP(A2:A50, LAMBDA(x, UPPER(TRIM(x))))
```

**说明**：对 A 列每个单元格先去除前后空格（`TRIM`），再转为大写（`UPPER`），结果自动溢出填充。

---

## 示例 2 ⭐⭐ — 双数组运算：计算单价 × 数量

```excel
=MAP(B2:B50, C2:C50, LAMBDA(单价, 数量, 单价 * 数量))
```

| 参数 | 说明 |
|------|------|
| `B2:B50` | 单价列 |
| `C2:C50` | 数量列 |

**说明**：两个数组必须行数相同。MAP 会逐行取对应元素传入 LAMBDA。

---

## 示例 3 ⭐⭐⭐ — 条件标记：给成绩打等级

```excel
=MAP(D2:D100, LAMBDA(score,
  IF(score>=90, "A",
    IF(score>=80, "B",
      IF(score>=70, "C",
        IF(score>=60, "D", "F")
      )
    )
  )
))
```

**说明**：对每个分数进行多级判断，返回等级字母。MAP 保证逐元素处理，不会混淆行与列。

---

## 示例 4 ⭐⭐⭐⭐ — 文本解析：提取邮箱域名

```excel
=MAP(A2:A200, LAMBDA(email,
  LET(
    pos, FIND("@", email),
    MID(email, pos+1, LEN(email)-pos)
  )
))
```

**说明**：在 LAMBDA 内使用 `LET` 缓存 `@` 位置，然后用 `MID` 截取域名部分。如果数据可能不含 `@`，需要增加 `IFERROR` 保护。

---

## 示例 5 ⭐⭐⭐⭐⭐ — 三数组联动 + 复杂逻辑：运费计算

```excel
=MAP(B2:B100, C2:C100, D2:D100,
  LAMBDA(重量, 距离, 会员,
    LET(
      基础费, IF(重量<=1, 8, 8 + CEILING(重量-1, 1)*3),
      距离系数, IF(距离>500, 1.5, 1),
      折扣, IF(会员="是", 0.85, 1),
      ROUND(基础费 * 距离系数 * 折扣, 2)
    )
  )
)
```

| 参数 | 说明 |
|------|------|
| `B列` | 包裹重量（kg） |
| `C列` | 运输距离（km） |
| `D列` | 是否会员（"是"/"否"） |

**说明**：MAP 最多可接收多个数组。每个 LAMBDA 参数对应一个数组的当前元素，适合多条件联合计算。

---

## 应用注意点

| 要点 | 说明 |
|------|------|
| 数组尺寸 | 所有输入数组的行列数必须一致，否则返回 `#VALUE!` |
| 返回值 | LAMBDA 对每个元素只能返回**单个值**，不能返回数组 |
| 性能 | 大数据量（>10,000 行）时 MAP 可能较慢，考虑用原生数组公式替代 |
| 错误处理 | 建议在 LAMBDA 内部用 `IFERROR` 或 `IF(ISERROR(...))` 保护异常值 |
| 与 BYROW 区分 | MAP 是逐**元素**处理；BYROW 是逐**行**处理（行可以有多列） |
