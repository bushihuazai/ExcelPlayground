# 15 — 综合实战流水线：多函数组合构建数据处理管道

> **适用版本**：Microsoft 365 / Excel 2021+

## 简介

前面 14 个场景各自展示了单个函数的能力。真正的威力在于**组合**——用 `LET` 定义管道，将 `FILTER`、`SORT`、`UNIQUE`、`MAP`、`REDUCE`、`VSTACK`、`HSTACK` 串联起来，一个公式完成从清洗到输出的全流程。

---

## 示例 1 ⭐ — 筛选 + 排序：最基本的两步管道

```excel
=SORT(
  FILTER(A2:E100, C2:C100="华东", "无数据"),
  5, -1
)
```

**说明**：先筛选华东区数据，再按第 5 列（销售额）降序排列。两个函数嵌套，读起来像 SQL 的 `WHERE ... ORDER BY`。

---

## 示例 2 ⭐⭐ — 去重 + 查找：生成维度映射表

```excel
=LET(
  prodID,   B2:B500,
  prodName, XLOOKUP(UNIQUE(prodID), Sheet2!A:A, Sheet2!B:B, "未知"),
  HSTACK(UNIQUE(prodID), prodName)
)
```

**说明**：

1. `UNIQUE` 提取所有不重复的产品 ID
2. `XLOOKUP` 批量查找产品名称（XLOOKUP 天然支持数组输入）
3. `HSTACK` 拼成两列映射表

---

## 示例 3 ⭐⭐⭐ — 分组统计：纯公式实现透视表

```excel
=LET(
  raw,      A2:E500,
  region,   INDEX(raw,,2),
  amount,   INDEX(raw,,5),
  u,        SORT(UNIQUE(FILTER(region, region<>""))),
  sumCol,   MAP(u, LAMBDA(r, SUM(FILTER(amount, region=r)))),
  cntCol,   MAP(u, LAMBDA(r, ROWS(FILTER(amount, region=r)))),
  avgCol,   MAP(u, LAMBDA(r, ROUND(AVERAGE(FILTER(amount, region=r)),2))),
  maxCol,   MAP(u, LAMBDA(r, MAX(FILTER(amount, region=r)))),
  header,   {"区域","合计","笔数","均值","最大值"},
  body,     HSTACK(u, sumCol, cntCol, avgCol, maxCol),
  VSTACK(header, SORT(body, 2, -1))
)
```

**说明**：

1. 提取并排序唯一区域
2. 用 MAP 分别计算 4 个指标
3. HSTACK 横向拼接 → VSTACK 添加表头
4. 最终按合计降序排列

等价于数据透视表，但完全由公式驱动，数据更新时自动刷新。

## 示例 4 ⭐⭐⭐⭐ — ETL 管道：清洗 → 转换 → 汇总 → 输出

```excel
=LET(
  /* ① 提取原始数据 */
  raw,       RawData!A2:G1000,

  /* ② 清洗：去掉空行和无效金额 */
  cleaned,   FILTER(raw,
               (INDEX(raw,,1)<>"") *
               ISNUMBER(INDEX(raw,,6)) *
               (INDEX(raw,,6)>0)
             ),

  /* ③ 转换：添加计算列（利润率） */
  revenue,   INDEX(cleaned,,6),
  cost,      INDEX(cleaned,,7),
  margin,    MAP(revenue, cost, LAMBDA(r,c, ROUND((r-c)/r*100, 1))),
  enriched,  HSTACK(cleaned, margin),

  /* ④ 汇总：按部门分组 */
  dept,      INDEX(enriched,,3),
  u,         SORT(UNIQUE(dept)),
  deptAvgM,  MAP(u, LAMBDA(d,
               ROUND(AVERAGE(FILTER(margin, dept=d)), 1)
             )),
  deptSum,   MAP(u, LAMBDA(d,
               SUM(FILTER(revenue, dept=d))
             )),

  /* ⑤ 输出：拼装报表 */
  header,    {"部门","总收入","平均利润率%"},
  body,      HSTACK(u, deptSum, deptAvgM),
  VSTACK(header, SORT(body, 2, -1))
)
```

**说明**：

| 步骤 | 功能 | 核心函数 |
|------|------|----------|
| ① | 引用原始数据 | 工作表引用 |
| ② | 清洗无效行 | FILTER + ISNUMBER |
| ③ | 添加利润率列 | MAP + HSTACK |
| ④ | 分组汇总 | UNIQUE + MAP + FILTER |
| ⑤ | 拼装输出 | HSTACK + VSTACK + SORT |

整个流程在一个 `LET` 中完成，每步有明确命名，维护修改都很方便。

## 示例 5 ⭐⭐⭐⭐⭐ — 完整方案：动态仪表板（含交互筛选）

假设 Z1 为区域下拉框、Z2 为日期起始、Z3 为日期结束。

```excel
=LET(
  /* 数据源 */
  raw,      Data!A2:H5000,
  date,     INDEX(raw,,1),
  region,   INDEX(raw,,3),
  product,  INDEX(raw,,4),
  qty,      INDEX(raw,,6),
  revenue,  INDEX(raw,,7),

  /* 动态筛选 */
  mask, (IF(Z1="全部", TRUE, region=Z1)) *
        (date>=Z2) * (date<=Z3) *
        (revenue>0),
  filtered, FILTER(raw, mask, "无匹配数据"),

  /* 分组：按产品汇总 */
  fProd,    INDEX(filtered,,4),
  fQty,     INDEX(filtered,,6),
  fRev,     INDEX(filtered,,7),
  u,        SORT(UNIQUE(fProd)),
  sumQty,   MAP(u, LAMBDA(p, SUM(FILTER(fQty, fProd=p)))),
  sumRev,   MAP(u, LAMBDA(p, SUM(FILTER(fRev, fProd=p)))),
  avgPrice, MAP(u, LAMBDA(p,
              ROUND(SUM(FILTER(fRev, fProd=p)) / SUM(FILTER(fQty, fProd=p)), 2)
            )),

  /* 排名列 */
  rankCol,  MAP(sumRev, LAMBDA(s, RANK(s, sumRev))),

  /* 输出 */
  header, {"排名","产品","总数量","总收入","平均单价"},
  body,   HSTACK(rankCol, u, sumQty, sumRev, avgPrice),
  report, VSTACK(header, SORT(body, 1, 1)),

  report
)
```

**说明**：

1. **交互筛选**：区域下拉 + 日期范围，用户修改即刻刷新
2. **分组聚合**：按产品维度计算数量、收入、均价
3. **排名**：`RANK` 函数生成排名列
4. **输出**：带表头的完整排行榜

这是一个"纯公式驱动的动态仪表板"——无 VBA、无数据透视表、无 Power Query，全部逻辑写在一个单元格里。

---

## 应用注意点

| 要点 | 说明 |
|------|------|
| 用 LET 命名每步 | 每个中间结果都应有清晰的变量名，方便维护 |
| 先简后繁 | 先写核心两三步验证逻辑正确，再逐步添加更多步骤 |
| FILTER 空值保护 | 始终提供 `if_empty` 参数，避免级联错误 |
| 性能 | 大数据量（>10,000 行）时，嵌套 MAP + FILTER 可能较慢，可考虑 SUMIFS 替代 |
| 调试技巧 | 将 `LET` 最后的返回值临时替换为某个中间变量，可逐步验证每一步的结果 |
| 团队协作 | 将常用的管道步骤封装为命名 LAMBDA 函数，提升团队复用率 |
