# 14 — XLOOKUP 高级查找：替代 VLOOKUP 的全能方案

> **适用版本**：Microsoft 365 / Excel 2021+

## 简介

`XLOOKUP` 是 VLOOKUP / HLOOKUP / INDEX+MATCH 的升级替代。支持精确/近似/通配符匹配，支持反向查找，无需关心列号。

语法：

```
=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])
```

| 参数 | 说明 |
|------|------|
| `match_mode` | 0=精确，-1=精确或次小，1=精确或次大，2=通配符 |
| `search_mode` | 1=从头，-1=从尾，2=二分升序，-2=二分降序 |

---

## 示例 1 ⭐ — 精确查找：根据工号查姓名

```excel
=XLOOKUP(G2, A2:A500, B2:B500, "未找到")
```

**说明**：在 A 列中查找 G2 的值，返回对应 B 列值。第四参数 `"未找到"` 避免 `#N/A` 错误。

---

## 示例 2 ⭐⭐ — 返回多列：查找完整记录

```excel
=XLOOKUP(G2, A2:A500, B2:E500, "无此记录")
```

**说明**：`return_array` 可以是多列区域，一次返回该行的 B–E 列所有数据。这是 VLOOKUP 做不到的。

---

## 示例 3 ⭐⭐⭐ — 近似匹配：税率区间查找

税率表（F 列为下限，G 列为税率）：

| 下限 F | 税率 G |
|--------|--------|
| 0 | 3% |
| 3000 | 10% |
| 12000 | 20% |
| 25000 | 25% |

```excel
=XLOOKUP(H2, F2:F5, G2:G5, , -1)
```

**说明**：`match_mode = -1` 表示"精确匹配或次小值"。当 H2=15000 时，匹配到 12000 这一档，返回 20%。

> ⚠️ 使用近似匹配时，lookup_array **必须升序排列**（除非使用二分搜索模式）。

---

## 示例 4 ⭐⭐⭐⭐ — 通配符匹配：模糊查找

```excel
=XLOOKUP("*科技*", B2:B200, C2:C200, "未找到", 2)
```

**说明**：`match_mode = 2` 启用通配符。`*` 匹配任意字符，`?` 匹配单个字符。返回第一个公司名包含"科技"的记录。

---

## 示例 5 ⭐⭐⭐⭐⭐ — 嵌套 XLOOKUP：二维交叉查找

假设有一张"产品 × 月份"的销售额矩阵：

- A 列：产品名称（A2:A20）
- 第 1 行：月份（B1:M1）
- 数据区域：B2:M20

根据指定的产品（P1）和月份（P2）查找销售额：

```excel
=XLOOKUP(P1, A2:A20,
  XLOOKUP(P2, B1:M1, B2:M20)
)
```

**说明**：

1. **内层** `XLOOKUP(P2, B1:M1, B2:M20)` — 在列标题中查找月份，返回该月份对应的整列数据（20 行）
2. **外层** `XLOOKUP(P1, A2:A20, 内层结果)` — 在产品列中查找产品名，从内层返回的列中取出对应行的值

这等价于 `INDEX + MATCH + MATCH` 的二维查找，但语义更清晰。

---

## 应用注意点

| 要点 | 说明 |
|------|------|
| 替代 VLOOKUP | XLOOKUP 无需列号，可向左查找，功能全面优于 VLOOKUP |
| if_not_found | 始终建议提供此参数，避免 `#N/A` 传播到下游公式 |
| 多列返回 | return_array 可以是多列甚至整行，一次获取多个字段 |
| 搜索方向 | `search_mode = -1` 从尾搜索，可用于查找最后一条匹配记录 |
| 二分查找 | 对大数据集（>10,000 行），使用 `search_mode = 2/-2` 配合排序数据可大幅提升性能 |
