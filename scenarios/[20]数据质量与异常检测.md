# 20 — 数据质量与异常检测：验证、清洗与质量评分的纯公式方案

> **适用版本**：Microsoft 365 / Excel 2021+

## 简介

数据质量是一切分析的基础。本场景综合运用 `LET`、`MAP`、`FILTER`、`BYROW`、`REDUCE`、`TEXTJOIN`、`HSTACK`、`VSTACK` 等函数，构建从单字段校验到全表质量评分的完整数据验证体系——发现问题行、标记异常、生成质量报告。

假设待验证数据 `A2:F500`，列依次为：**姓名(A)、手机号(B)、邮箱(C)、金额(D)、日期(E)、部门(F)**。

---

## 示例 1 ⭐ — 基础完整性检查：标记空值与缺失

```excel
=LET(
  data,    A2:F500,
  n,       ROWS(data),
  headers, {"姓名","手机号","邮箱","金额","日期","部门"},
  n_cols,  COLUMNS(data),

  /* 逐行统计空字段数 */
  empty_count, BYROW(data, LAMBDA(r,
    SUM(MAP(r, LAMBDA(v, IF(OR(v="", ISBLANK(v)), 1, 0))))
  )),

  /* 逐列统计非空率 */
  col_filled, BYCOL(data, LAMBDA(c,
    ROUND(SUM(MAP(c, LAMBDA(v, IF(AND(v<>"", NOT(ISBLANK(v))), 1, 0)))) / n * 100, 1)
  )),

  /* 完整性报告 */
  col_report, VSTACK(
    {"字段","非空率%"},
    HSTACK(TOROW(headers), TOROW(col_filled))
  ),

  /* 不完整的行 */
  row_idx, SEQUENCE(n),
  bad_rows, FILTER(
    HSTACK(row_idx + 1, data, empty_count),
    empty_count > 0,
    "所有记录完整 ✓"
  ),

  VSTACK(
    {"【字段完整性】","","","","","",""},
    HSTACK(headers, "", "非空率%"),
    HSTACK(headers, "", col_filled & "%"),
    {"","","","","","",""},
    {"【缺失记录明细】（行号）","姓名","手机号","邮箱","金额","日期","空字段数"},
    bad_rows
  )
)
```

**说明**：

1. `BYROW` 逐行统计空字段数——快速识别不完整记录
2. `BYCOL` 逐列计算非空率——了解哪些字段缺失最严重
3. `FILTER` 输出所有有缺失的行，附带行号方便定位

## 示例 2 ⭐⭐ — 格式校验：手机号、邮箱、数值范围

```excel
=LET(
  data,   A2:F500,
  n,      ROWS(data),
  name,   INDEX(data,,1),
  phone,  INDEX(data,,2),
  email,  INDEX(data,,3),
  amount, INDEX(data,,4),
  dt,     INDEX(data,,5),
  row_id, SEQUENCE(n) + 1,

  /* 手机号校验：11 位数字且以 1 开头 */
  phone_ok, MAP(phone, LAMBDA(p,
    AND(LEN(TEXT(p, "0")) = 11, LEFT(TEXT(p, "0"), 1) = "1", ISNUMBER(p + 0))
  )),

  /* 邮箱校验：包含 @ 和 . */
  email_ok, MAP(email, LAMBDA(e,
    AND(ISNUMBER(FIND("@", e)), ISNUMBER(FIND(".", e)), LEN(e) >= 5)
  )),

  /* 金额校验：数值型且 >= 0 */
  amount_ok, MAP(amount, LAMBDA(a, AND(ISNUMBER(a), a >= 0))),

  /* 日期校验：有效日期且不超过今天 */
  date_ok, MAP(dt, LAMBDA(d, AND(ISNUMBER(d), d >= DATE(2000,1,1), d <= TODAY()))),

  /* 错误描述 */
  errors, MAP(row_id, phone_ok, email_ok, amount_ok, date_ok,
    LAMBDA(i, p, e, a, d,
      LET(
        msgs, TEXTJOIN("; ", TRUE,
          IF(NOT(p), "手机号格式错误", ""),
          IF(NOT(e), "邮箱格式错误", ""),
          IF(NOT(a), "金额异常", ""),
          IF(NOT(d), "日期无效", "")
        ),
        IF(msgs <> "", "行" & i & ": " & msgs, "")
      )
    )
  ),

  result, FILTER(errors, errors <> "", "所有记录格式正确 ✓"),

  VSTACK(
    {"【格式校验结果】"},
    {"通过率: " & ROUND(SUM(phone_ok*email_ok*amount_ok*date_ok)/n*100, 1) & "%"},
    {""},
    {"错误明细"},
    result
  )
)
```

**说明**：

1. 手机号：验证长度为 11 且以 1 开头（中国手机号规则）
2. 邮箱：验证包含 `@` 和 `.`（基础格式检查）
3. 金额：验证为非负数值
4. 日期：验证在合理范围内（2000年至今）

> 备选：将手机号规则改为 `LEN=10` 并去掉首位检查可适配其他国家号码。

## 示例 3 ⭐⭐⭐ — 重复记录检测与标记

```excel
=LET(
  data,   A2:F500,
  n,      ROWS(data),
  name,   INDEX(data,,1),
  phone,  INDEX(data,,2),

  /* 以姓名+手机号作为唯一键 */
  key,    MAP(name, phone, LAMBDA(na, ph, na & "|" & TEXT(ph, "0"))),

  /* 每条记录出现次数 */
  dup_count, MAP(key, LAMBDA(k, SUM((key = k) * 1))),

  /* 标记首次出现 vs 重复 */
  first_occur, MAP(SEQUENCE(n), LAMBDA(i,
    LET(
      current_key, INDEX(key, i),
      prev_keys, IF(i = 1, "", INDEX(key, SEQUENCE(i - 1))),
      IF(i = 1, TRUE, ISERROR(MATCH(current_key, prev_keys, 0)))
    )
  )),

  tag, MAP(dup_count, first_occur, LAMBDA(dc, fo,
    IF(dc = 1, "",
      IF(fo, "首次(" & dc & "条)", "⚠️重复"))
  )),

  /* 输出重复记录 */
  has_dup, (dup_count > 1),
  result,  FILTER(
    HSTACK(SEQUENCE(n) + 1, data, dup_count, tag),
    has_dup,
    "未发现重复记录 ✓"
  ),

  VSTACK(
    {"【重复检测】"},
    {"唯一记录数: " & ROWS(UNIQUE(key)) & " / 总记录数: " & n},
    {"重复组数: " & ROWS(UNIQUE(FILTER(key, dup_count > 1, "")))},
    {""},
    {"行号","姓名","手机号","邮箱","金额","日期","部门","重复次数","标记"},
    result
  )
)
```

**说明**：

1. 将姓名+手机号拼接为唯一键
2. `MAP + SUM` 统计每个键出现的次数
3. 区分"首次出现"和"重复"，方便决定保留哪条

> 备选：修改 `key` 的拼接逻辑（如只用手机号或邮箱）可切换去重维度。

## 示例 4 ⭐⭐⭐⭐ — 统计离群值检测（Z-Score 与 IQR 方法）

```excel
=LET(
  data,   A2:F500,
  amount, INDEX(data,,4),
  name,   INDEX(data,,1),
  n,      ROWS(data),

  /* 过滤有效数值 */
  valid,   ISNUMBER(amount),
  v_amt,   FILTER(amount, valid),

  /* Z-Score 方法 */
  mu,      AVERAGE(v_amt),
  sigma,   STDEV.S(v_amt),
  z_score, MAP(amount, LAMBDA(a,
    IF(ISNUMBER(a), ROUND(ABS((a - mu) / MAX(sigma, 0.001)), 2), "")
  )),
  z_outlier, MAP(z_score, LAMBDA(z,
    IF(z = "", "", IF(z > 3, "🔴极端", IF(z > 2, "🟡可疑", "")))
  )),

  /* IQR 方法 */
  q1,       PERCENTILE.INC(v_amt, 0.25),
  q3,       PERCENTILE.INC(v_amt, 0.75),
  iqr,      q3 - q1,
  lower,    q1 - 1.5 * iqr,
  upper,    q3 + 1.5 * iqr,
  iqr_flag, MAP(amount, LAMBDA(a,
    IF(NOT(ISNUMBER(a)), "",
      IF(OR(a < lower, a > upper), "🔴超出IQR", ""))
  )),

  /* 综合标记 */
  combined, MAP(z_outlier, iqr_flag, LAMBDA(z, i,
    TEXTJOIN(" ", TRUE, z, i)
  )),
  is_outlier, MAP(combined, LAMBDA(c, TRIM(c) <> "")),

  /* 输出 */
  stats, VSTACK(
    {"【统计摘要】",""},
    HSTACK("均值", ROUND(mu, 2)),
    HSTACK("标准差", ROUND(sigma, 2)),
    HSTACK("Q1 / Q3", q1 & " / " & q3),
    HSTACK("IQR", ROUND(iqr, 2)),
    HSTACK("IQR 正常范围", ROUND(lower, 2) & " ~ " & ROUND(upper, 2)),
    {"",""}
  ),

  outliers, FILTER(
    HSTACK(SEQUENCE(n) + 1, name, amount, z_score, combined),
    is_outlier,
    {"未发现异常值 ✓","","","",""}
  ),

  VSTACK(
    stats,
    {"【异常值明细】","","","",""},
    {"行号","姓名","金额","Z-Score","异常标记"},
    outliers
  )
)
```

**说明**：

1. **Z-Score 方法**：|Z| > 3 为极端异常，>2 为可疑
2. **IQR 方法**：低于 Q1−1.5×IQR 或高于 Q3+1.5×IQR 的为异常值
3. 两种方法并行，综合标记提升检出准确率

> 备选：修改 Z-Score 阈值（如从 3 改为 2.5）可调整敏感度。
> 备选：将 `amount` 替换为其他数值列即可检测不同字段。

## 示例 5 ⭐⭐⭐⭐⭐ — 全表数据质量评分卡

```excel
=LET(
  data,   A2:F500,
  n,      ROWS(data),
  name,   INDEX(data,,1),
  phone,  INDEX(data,,2),
  email,  INDEX(data,,3),
  amount, INDEX(data,,4),
  dt,     INDEX(data,,5),
  dept,   INDEX(data,,6),
  headers, {"姓名","手机号","邮箱","金额","日期","部门"},

  /* ① 完整性评分 */
  filled, BYCOL(data, LAMBDA(c,
    SUM(MAP(c, LAMBDA(v, IF(AND(v<>"", NOT(ISBLANK(v))), 1, 0)))) / n
  )),
  completeness, ROUND(AVERAGE(filled) * 100, 1),

  /* ② 格式正确性 */
  phone_ok, SUM(MAP(phone, LAMBDA(p,
    IF(AND(LEN(TEXT(p,"0"))=11, LEFT(TEXT(p,"0"),1)="1"), 1, 0)
  ))) / n,
  email_ok, SUM(MAP(email, LAMBDA(e,
    IF(AND(ISNUMBER(FIND("@",e)), ISNUMBER(FIND(".",e))), 1, 0)
  ))) / n,
  amount_ok, SUM(MAP(amount, LAMBDA(a, IF(AND(ISNUMBER(a), a>=0), 1, 0)))) / n,
  date_ok, SUM(MAP(dt, LAMBDA(d, IF(AND(ISNUMBER(d), d>=DATE(2000,1,1), d<=TODAY()), 1, 0)))) / n,
  validity, ROUND((phone_ok + email_ok + amount_ok + date_ok) / 4 * 100, 1),

  /* ③ 唯一性 */
  key,         MAP(name, phone, LAMBDA(na, ph, na & "|" & ph)),
  unique_rate, ROUND(ROWS(UNIQUE(key)) / n * 100, 1),

  /* ④ 异常值比例 */
  v_amt,    FILTER(amount, ISNUMBER(amount)),
  q1,       PERCENTILE.INC(v_amt, 0.25),
  q3,       PERCENTILE.INC(v_amt, 0.75),
  iqr,      q3 - q1,
  outliers, SUM(MAP(amount, LAMBDA(a,
    IF(AND(ISNUMBER(a), OR(a < q1 - 1.5*iqr, a > q3 + 1.5*iqr)), 1, 0)
  ))),
  outlier_rate, ROUND((1 - outliers / MAX(SUM(ISNUMBER(amount)*1), 1)) * 100, 1),

  /* ⑤ 综合质量分 (加权) */
  w_complete,  0.30,
  w_validity,  0.30,
  w_unique,    0.20,
  w_outlier,   0.20,
  total_score, ROUND(
    completeness * w_complete +
    validity * w_validity +
    unique_rate * w_unique +
    outlier_rate * w_outlier,
    1
  ),

  /* 等级评定 */
  grade, IF(total_score >= 90, "A — 优秀",
           IF(total_score >= 75, "B — 良好",
             IF(total_score >= 60, "C — 一般", "D — 需改进"))),

  /* ⑥ 各字段明细 */
  field_detail, VSTACK(
    {"字段", "非空率%", "备注"},
    HSTACK("姓名", ROUND(INDEX(filled,,1)*100,1) & "%", ""),
    HSTACK("手机号", ROUND(INDEX(filled,,2)*100,1) & "%", "格式正确率: " & ROUND(phone_ok*100,1) & "%"),
    HSTACK("邮箱", ROUND(INDEX(filled,,3)*100,1) & "%", "格式正确率: " & ROUND(email_ok*100,1) & "%"),
    HSTACK("金额", ROUND(INDEX(filled,,4)*100,1) & "%", "合规率: " & ROUND(amount_ok*100,1) & "%"),
    HSTACK("日期", ROUND(INDEX(filled,,5)*100,1) & "%", "有效率: " & ROUND(date_ok*100,1) & "%"),
    HSTACK("部门", ROUND(INDEX(filled,,6)*100,1) & "%", "")
  ),

  /* ⑦ 输出评分卡 */
  VSTACK(
    {"══════════════════════════","",""},
    {"  数据质量评分卡","",""},
    {"══════════════════════════","",""},
    {"","",""},
    {"维度","得分","权重"},
    HSTACK("完整性", completeness & "%", w_complete * 100 & "%"),
    HSTACK("格式正确性", validity & "%", w_validity * 100 & "%"),
    HSTACK("唯一性", unique_rate & "%", w_unique * 100 & "%"),
    HSTACK("异常值控制", outlier_rate & "%", w_outlier * 100 & "%"),
    {"","",""},
    HSTACK("综合评分", total_score, ""),
    HSTACK("质量等级", grade, ""),
    {"","",""},
    field_detail
  )
)
```

**说明**：

| 维度 | 计算方式 | 权重 |
|------|----------|------|
| 完整性 | 各字段非空率的平均值 | 30% |
| 格式正确性 | 手机号/邮箱/金额/日期校验通过率均值 | 30% |
| 唯一性 | 去重后记录数 / 总记录数 | 20% |
| 异常值控制 | 1 − 离群值占比 | 20% |

加权得出综合质量分（满分 100），并划分 A/B/C/D 等级。

> 备选：修改各维度权重 `w_complete` 等可适配不同业务对数据质量的侧重点。
> 备选：新增"时效性"维度（如最新记录距今天数）可扩展为 5 维评分。

## 应用注意点

| 要点 | 说明 |
|------|------|
| 校验规则 | 手机号、邮箱等格式规则应根据实际业务调整；Excel 正则能力有限，复杂规则需分步验证 |
| 阈值设定 | Z-Score 和 IQR 阈值需根据数据分布特点选择；偏态分布建议优先使用 IQR 方法 |
| 空值 vs 零值 | 区分"真空"和"填零"——某些场景下 0 也是有效数据 |
| 性能 | 全表扫描（500+ 行 × 6 列）的 MAP 嵌套可能需要 2~5 秒计算；行数更多时考虑分批处理 |
| 持续监控 | 建议将质量评分卡公式设为固定报表，每次数据更新后自动刷新分数 |
| 扩展 | 可将各校验逻辑封装为命名函数（如 `VALIDATE_PHONE`、`VALIDATE_EMAIL`），提升团队复用率 |
